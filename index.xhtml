What's Linux?

  • Linux is a kernel, a software ecosystem and an attitude
  • Modern science relies on and is influenced by all three parts
  • Without Linux, it would be much harder and more expensive to:
    • Sequence genomes
    • Scan the sky
    • Search for the Higgs

What's a Higgs boson and why should we care?

  • Higgs Boson: subatomic particle
  • Proposed in 1964 by Peter Higgs, not yet observed in experiments
  • Fundamental particle expected by the theories we use to understand particle physics
  • Occur when other particles become massive (Higgs mechanism)
  • We're pretty sure about the mechanism, so we think we should be able to see the Higgs itself
  • This part of the universe is very...statistical
  • Finding the Higgs would solidify this model even further
  • Not finding the Higgs would provoke lots of very interesting physics

How do you find a Higgs?

Experimental particle physics in six easy steps:

  1. Build a particle accelerator, smash particles together and gather data
  2. Throw as much of that data away as you can
  3. Store the rest
  4. Analyze the data
  5. ...
  6. Nobel prize!

LHC: Large Hadron Collider

  • Global collaboration built the LHC accelerator at CERN
    • CERN: European Center for Nuclear Research, Geneva, Switzerland
  • Thousands of physicists, students, engineers (real and otherwise)
  • LHC accelerates protons around a ring as long as a marathon
  • Particles approach speed of light and smash into each other
  • Detectors gather data

CMS: Compact Muon Solenoid

  • Four main detectors at LHC: CMS, ATLAS, LHCb, Alice
  • Detector collaborations compete with each other, verify findings
  • Detectors are like huge digital cameras
    • Millions of silicon and optical sensors wrapped around the accelerator tube
    • Gather evidence (particle tracks, energy) to identify what was produced in collision

Sorting the unknown from the boring

  • Most of what is produced in a collision is well-understood (ie. boring)
  • Each collision generates 1 MB of data; collisions happen 30,000,000 times a second
  • Can't store or process all of that data, so we throw most of it out
  • 76 million channels in CMS send data to:
  • Fast hardware triggers: 1000x reduction
    • Custom FPGAs (designed and built at Wisconsin for CMS)
    • Send 50,000 events per second by fiber to software triggers
  • High Level trigger: another 1000x reduction
    • Software selectors (some also built at Wisconsin) run on a Linux cluster
    • Archive 100 events per second to tape, 50ms per selection

  • CERN stores a copy of all data to tape
    • Tape is cheap, reliable and doesn't require expensive electricity
  • Data moves from CERN to network of national labs, stored on tape again
    • In the US, Fermilab, Brookhaven
    • Europe/Russia, Asia, South America, ...
  • Universities request datasets and begin analysis
    • Caltech, MIT, Nebraska, Purdue, UC San Diego, Vanderbilt, Wisconsin
    • Data lives on disks

Searching for the Higgs at Wisconsin

  • We run Linux on hundreds of commodity servers
    • Perform thousands of CPU hours of computation each day
    • Store hundreds of terabytes
  • We need Linux because...
    • Can use the source to debug, optimize
    • Rely on the large community for support
    • Open source software stack
    • It's free

Linux helps us take care of the easy stuff

  • Kickstart/Yum/Cfengine/ssh for mass deployment and maintenance
  • Mysql, Postgresql, Redis, BDB for (sometimes) relational data
  • Python, Shell, Perl, Java for custom software
  • Nagios, cron, collaboration-specific central monitoring to track performance
  • AFS, sendmail, named, httpd, (r)syslog for everyday infrastructure

Analysis

  • Condor: batch system for high-througput computing, developed at Wisconsin
  • Jobs submitted by hundreds of users from around the world; Condor matches tasks to CPU cores
  • Transparently schedule jobs to "scavenge" idle time on campus computer labs, other clusters
  • Thousands of jobs/day, tens of thousands of CPU hours/day
  • From 4 to 24 cores, 4-48 GB RAM

Storage

  • dCache: distributed file system (Java)
  • Stores data per file (not block), custom Python software to replicate
  • Namespace lookups are expensive
  • Lots of overhead per transfer, large files are more efficient
  • Commodity servers with directly attached disks (1U, 2 to 4 SATA, .5 to 2 TB)
    • These servers run analysis and serve data
  • Some servers with dedicated RAID (4U, 24 - 32 SATA, 1 - 2 TB)

What are we working on now?

  • Scale computing and storage every year
  • Speed up data access
  • Reduce cost of colocated storage/computing
  • Harden and optimize database services
  • Simplify architecture
  • In-depth monitoring, history
  • Test-driven operations/deployment
  • Find a way to grow past heat limits

Plans

  • Replace proprietary storage cluster with Hadoop file system
    • Migrate .5 PB of data with minimal interruption
  • Zero-knowledge data access: CMS xrootd redirector
    • Multiple copies of data across the US
    • If one datacenter's storage fails, access nearest available copy
    • View events from their laptops without knowing file location
  • Abstract job submission: GlideinWMS
    • Factories send pilot jobs that themselves submit actual physics tasks
    • Easier to handle failure, balance resources
    • Makes authentication/authorization, security, accounting harder
